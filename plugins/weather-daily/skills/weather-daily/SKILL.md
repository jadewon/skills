# Weather Daily

매일 아침 8시경, 사용자가 사는 지역의 오늘 날씨를 조사해 **한 줄 요약 + 행동 가능한 조언**을 Slack 채널에 보낸다.

설정값(서비스키·좌표)은 `.env` 에서 로드한다. `.env.example` 참고. Slack 전송은 webhook 이 아니라 연결된 **Slack 커넥터**로 한다.

## 절대 규칙

- 코드베이스 탐색/구조 파악을 하지 마라. 아래 순서만 정확히 따라라.
- **수치는 둘째 줄 요약 한 곳에만.** 조언 줄 본문에 "15℃", "60%" 등 재노출 금지 ("쌀쌀해요" 식).
- **"좋음/보통"은 노이즈.** 출력하지 않는다. 챙길 게 없으면 요약 + 옷차림만으로 끝낸다.
- 인사말/마무리 멘트 금지. 본론만.
- *bold* (single asterisk), `code` 이외 마크다운 금지 (`**`, `#`, `[text](url)`, ` ``` ` 금지). `~` 은 Slack 취소선(`~text~`) 용도로만 금지 — 범위 구분(`16~22℃`) 은 허용.

## 실행 순서

### 1. 공휴일 확인

```bash
"${CLAUDE_SKILL_DIR}/weather-daily.sh" holiday
```

출력이 `true` 면 "공휴일 스킵" 출력 후 종료. Slack 전송하지 않는다.

### 2. 데이터 수집 + 집계

```bash
"${CLAUDE_SKILL_DIR}/weather-daily.sh" fetch
```

stdout 으로 아래 JSON 출력. 그대로 받아 분석:

```json
{
  "date": "20260512",
  "tmin": 16.0, "tmax": 22.0,
  "feel_min": 16.0, "feel_max": 22.0,
  "diurnal": 6.0,
  "rh_min": 45, "rh_max": 70,
  "wind_max": 1.8,
  "rain_hours": [
    {"time": "0600", "pop": 60, "pty": 1, "pcp": "1mm 미만"},
    {"time": "0700", "pop": 60, "pty": 1, "pcp": "2.0mm"}
  ],
  "pm10_seoul": "좋음",
  "pm25_seoul": "좋음",
  "uv_max_today": 5
}
```

`rain_hours` 의 `pty`: 0=없음, 1=비, 2=비/눈, 3=눈, 4=소나기.

### 3. 메시지 작성 (친근한 비서 톤)

위 JSON 을 보고 직접 작성. 규칙:

- **첫 줄**: `*5월 12일 화요일* — <한 줄 총평>`
- **둘째 줄 (요약 패널)**: `<tmin>~<tmax>℃ · 습도 <rh_min>~<rh_max>% · UV <uv_max_today>`
  - 가운뎃점 ` · ` 으로 구분. 다른 구분자 쓰지 마라.
  - `tmin == tmax` 면 단일 값 (`22℃`). 습도/UV 도 동일.
  - `uv_max_today` 가 0 이면 UV 칸 자체 생략 (야간/우천 시 0 자주 발생).
  - 이 줄에는 이모지 붙이지 마라.
- **셋째 블록**: 옷차림 추천 (체감 최저 기준, 아래 표) — 한 줄.
- **조건부 블록** (해당될 때만, 좋음/보통은 생략):
  - `rain_hours` 비어있지 않으면: 시간대 + 우산 안내
  - `diurnal ≥ 8` 또는 `wind_max ≥ 5`: 겉옷/체감 안내
  - `pm10_seoul` 또는 `pm25_seoul` 가 "나쁨" 이상: 마스크 안내
  - `uv_max_today ≥ 6`: 선크림 안내
- 조언 줄은 짧게. 둘째 줄에서 이미 노출한 수치를 본문에서 재사용 금지 (예: "16℃ 쌀쌀해요" X → "쌀쌀해요" O).
- 이모지는 줄 앞에 1개만. 본문 중간 이모지 금지.
- 매일 첫 줄 총평 표현을 다르게 — 같은 문장 반복 금지.

#### 옷차림 임계값 (체감 최저 기준)

| 체감 | 추천 |
|---|---|
| 28℃ 이상 | 반팔, 얇은 옷 |
| 23~27 | 반팔/얇은 셔츠 |
| 20~22 | 얇은 가디건/긴팔 |
| 17~19 | 얇은 니트/맨투맨 |
| 12~16 | 트렌치/얇은 자켓 |
| 9~11 | 자켓/야상 |
| 5~8 | 코트/가죽자켓 |
| 4 이하 | 패딩/두꺼운 코트 |

#### 미세먼지 멘트 (PM10/PM2.5 중 더 나쁜 쪽 기준 한 줄만)

- "나쁨" → `😷 미세먼지 나쁨, 마스크 챙기세요`
- "매우 나쁨" → `🚨 미세먼지 매우 나쁨, KF94 권장`

#### UV 멘트

- 6~7 → `🧴 자외선 강해요. 선크림 챙기세요`
- 8~10 → `☀️ 자외선 매우 강함 — 선크림+모자`
- 11+ → `⚠️ 자외선 위험 — 가급적 그늘로`

### 4. Slack 전송

작성한 메시지를 연결된 **Slack 커넥터** (`slack_send_message`) 로 채널 `C0B3747NKND` (#실험실-lab) 에 전송한다. webhook 직발송은 쓰지 않는다.

## 예시 출력

비 오는 흐린 봄날:

```
*5월 12일 화요일* — 흐리고 종일 비예보
16~22℃ · 습도 45~70% · UV 5

🧥 트렌치나 얇은 자켓이 적당해요
☂️ 출근길 비 와요. 오후 1~2시쯤 한 번 더 와요. 우산 챙기세요
```

화창한 따뜻한 날 (챙길 게 없을 때):

```
*5월 20일 화요일* — 화창하고 따뜻해요
18~24℃ · 습도 40~60% · UV 6

🧥 얇은 긴팔/가디건이면 충분해요
🧴 자외선 강해요. 선크림 챙기세요
```

## 셋업

1. `.env.example` 을 `.env` 로 복사
2. `KMA_SERVICE_KEY` 채움 (Slack 전송은 커넥터가 담당 — webhook 불필요)
3. `NX`, `NY`, `AREA_NO` 를 본인 지역으로 (기본값: 서울시청)
4. (선택) `HOLIDAY_API_URL` 설정
5. `chmod +x weather-daily.sh`
6. Slack 커넥터가 연결된 환경(로컬 Claude Code / claude.ai routine)에서 실행 — 대상 채널 `C0B3747NKND`

## 스케줄

발송이 LLM + Slack 커넥터 경로라 LLM 호출이 필요하다 (webhook 직발송 경로 제거됨). 머신이 꺼지면 miss 하는 로컬 cron 대신 **claude.ai routine (cloud)** 권장 — Slack 커넥터 연결 + Weekday 08:00 KST. 항상 켜진 서버에서 cron 으로 돌릴 경우:

```cron
0 8 * * 1-5 cd /path/to/skills/weather-daily && claude --print /weather-daily
```
